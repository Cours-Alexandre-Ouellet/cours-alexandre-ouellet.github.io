    Les ListView présentent les informations d'une liste dans l'interface. Elles possèdent plusieurs méthodes et propriétés pour gérer la sélection des éléments et afficher en temps réels les valeurs dans l'application.

    Dans cette page, l'exemple d'un système qui permet de gérer des employés est utilisé pour illustrer le comportement de la ListView.

    Peupler une ListView avec une ObservableList

    Afin de tirer profit des ListView, il est nécessaire d'utiliser les ObservableList plutôt que les ArrayList standards en java. Les ObservableList possèdent les mêmes méthodes et propriétés que l'ArrayList et implémentent à l'interne le patron observateur.

    Attention, les changements aux éléments dans la liste, eux ne font pas partie de l'observateur. Dans l'exemple, les champs de "employé" ne sont pas inclus dans la notification de l'observateur.

    @FXML
    private void initialize(){

        List<Employe> fixturesEmployes = new ArrayList<>();
        fixturesEmployes.add(new Employe("Bob", "Bobbidou", "Directeur", "../bob.png",
                LocalDate.of(1980, 5,16), 124471));
        fixturesEmployes.add(new Employe("Clara", "Lamontagne", "Gestionnaire de projet",
                "../clara.png", LocalDate.of(1974, 11, 22), 107412));

        // Crée un liste observable avec la méthode statique observableList de la classe FXCollections
        employes = FXCollections.observableList(fixturesEmployes);

        // setItems est la méthode pour assigner une ObservableList à une ListView et lier leurs observateurs  
        listeAfficherEmployes.setItems(employes);
    }
}

    Fabriquer ses propres éléments personnalisés

    Pour obtenir un affiche plus complet, on peut utiliser une classe dédiée pour fabriquer les objets. La classe doit hériter de ListCell et surcharger la méthode updateItem pour être utilisée. Il est aussi possible de définir l'affichage dans un fichier FXML, mais cela demande aussi de définir un contrôleur à la classe d'affichage pour gérer les éléments du FXML.

    Pour ce faire, on ajoute les classes suivantes :

    • ElementListeEmploye qui hérite de ListCell et permet de définir le comportement de l'item de liste;
    • ElementListeEmployeController qui gère l'affichage interne du contenu de l'item de la liste;

    Aussi, le fichier FXML elementlisteemploye.fxml est ajouté pour définir comment l'item est présenté dans la liste.

    package edu.cegepvicto.demoliste.elementliste;

import edu.cegepvicto.demoliste.Employe;
import javafx.fxml.FXMLLoader;
import javafx.scene.Node;
import javafx.scene.control.ListCell;

/**
 * Classe pour afficher un élément dans la liste view
 */
public class ElementListeEmploye extends ListCell<Employe> {

    private ElementListeEmployeController controleur;

    // Référence à l'élément chargé à la racine de la vue FXML
    private Node racine;

    public ElementListeEmploye() {
        super();

        try {
            FXMLLoader loader = new FXMLLoader(getClass().getResource("elementlisteemploye.fxml"));
            racine = loader.load();
            controleur = loader.getController();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public void updateItem(Employe employe, boolean empty) {
        super.updateItem(employe, empty);
        if (empty || employe == null) {
            setGraphic(null);
        } else {
            controleur.setEmploye(employe);
            setGraphic(racine);
        }
    }
}

    Sélectionner des éléments et accéder aux éléments sélectionnés

    La sélection des éléments se fait avec un SelectionModel qui est accessible par l'accesseur getSelectionModel de la ListView. Les principales méthodes du SelectionModel sont décrites dans le tableau ci-dessous.

    Tâche Méthode Description
    Sélectionner un élément à partir du code select Sélectionne l'indice ou l'élément passé en paramètre.
    Sélectionner tous les éléments selectAll
    Sélectionner le premier élément selectFirst
    Changer le mode de sélection (simple ou multiple) setSelectionMode Permet de sélectionner un seul ou plusieurs éléments, selon la valeur dans SELECTION_MODE.
    Récupérer l'élément sélectionné getSelectedItems Retourne une ObservableList des classes des éléments sélectionnés de la liste.
    Récupérer l'indice de l'élément sélectionné getSelectedIndices Retourne une ObservableList des indices des éléments sélectionnés de la liste.

    Réagir à la sélection d'éléments dans la liste

    Pour réagir à la sélection d'éléments, il faut ajouter un listener sur l'ObservableList des éléments sélectionnés dans le SelectionModel. Le listener prendra la forme d'une interface fonctionnelle (en java, les interfaces avec une seule méthode sont appelées interface fonctionnelle et peuvent être implémentées directement à l'aide d'un lambda).

    Le listener envoie un objet de type Change qui contient une liste de changement. Chaque changement contient l'information sur le retrait ou l'ajout d'éléments. Un remplacement d'un élément par un autre déclenche deux changements, d'où la boucle while pour parcourir tous les changements d'état. Selon la nature du changement, la méthode getAddedSubList et getRemoved permettent d'accéder aux éléments ajoutés et retirés.

    Dans le cas de l'exemple, comme on ne peut sélectionner qu'un élément, on utilise la méthode getFirst pour récupérer l'élément ajouté. Si le code du listener devient trop long, il est possible de le déplacer vers une classe.

    Réaction au changement d'élémemnt sélectionné
        private void initialize() {
        //...

        listeAfficherEmployes.getSelectionModel().getSelectedItems().addListener(
                    new ListChangeListener<Employe>() {
                        @Override
                        public void onChanged(Change<? extends Employe> change) {
                            while (change.next()) {
                                if (change.wasAdded()) {
                                    selectionnerEmploye(change.getAddedSubList().getFirst());
                                } else if (change.wasRemoved()) {
                                    selectionnerEmploye(null);
                                }
                            }
                        }
                    }
            );

        // Sélection par défaut de la première personne
        listeAfficherEmployes.getSelectionModel().selectFirst();
    }

    private void selectionnerEmploye(Employe employe){

        if(employeSelectionne == employe) {
            return;
        }

        effacerDonneesInfo();
        employeSelectionne = employe;
        afficherInfoEmploye(employeSelectionne);
    }

    Ajouter et retirer des éléments dans la liste

    Pour ajouter et retirer des éléments de la liste, il suffit de les ajouter ou de les retirer dans la ObservableList. Automatiquement (parce que c'est observable...), l'affichage se mettra à jour.

    Modification du contenu de la liste
        @FXML
    private void ajouterEmploye() {
        Employe nouvelEmploye = new Employe("", "", "À indiquer", LocalDate.now(), 0.0f);
        employes.add(nouvelEmploye);                    // Ajoute l'employé à la liste ET dans l'interface

        listeAfficherEmployes.getSelectionModel().selectLast();     // Sélectionne l'employé ajouté
    }

    @FXML
    private void supprimerEmploye() {
        if (employeSelectionne != null) {
            employes.remove(employeSelectionne);        // Retire l'employé de la liste ET de l'affichage
            listeAfficherEmployes.getSelectionModel().selectFirst();    // Sélectionne un emloyé valide
        }
    }
