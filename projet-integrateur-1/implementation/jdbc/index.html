    Dans cette section, nous verrrons comment connecter une base de données MySQL avec la bibliothèque Java Database Connectivity (JDBC). Le code de la section sur les architectures et l'exemple de système de voyage. Les modèle DAO est utilisé pour organiser le code. Bien qu'il ne soit pas nécessaire de l'utiliser, il aide particulièrement avec l'organisation du code.

    Installation de JDBC

    dependencies {
    //...

}

    Le connecteur peut changer selon le type de base de donnée utilisés.

    Concepts de base

    JDBC propose plusieurs interfaces pour manipuler les bases de données :

    • Connection : gère la connexion à la base de données;
    • PreparedStatement: permet de faire des opérations du CRUD avec une requête préparée (ce que vous devriez toujours faire);
    • ResultSet : contient les résultats de la requête (réponse du SGBD).

    Se connecter à la base de données

    Le code ci-dessous permet d'obtenir une connexion à la base de données. Notez qu'avoir plusieurs connexions simultanné risque de provoquer des problèmes,dont le fameux death-lock.

    Obtenir une connexion
        connexion = DriverManager.getConnection(

    connexion = DriverManager.getConnection(
        "jdbc:mysql://127.0.0.1:3306/nomBD", "root", "mysql"

    Évidemment, vous ne devriez pas passer directement les valeurs, mais plutôt utiliser des constantes ou un fichier de configuration.

    Le nom d'hôte

    Le nom d'hôte doit suivre le format suivant : jdbc:<type connecteur>://<ip>/<nomBD>

    Dans la requête préparée (ligne 7), les ? indique des valeurs à remplacer. Les lignes 11 et 12 affectent ses paramètres. D'autres méthodes existent selon le type de données comme setBool, setString, setFloat, setDouble, setDate, setTime, setTimestamp. Les paramètres sont numérotés dans l'ordre d'ajout en commençant à 1.

    Le prompt suivant vous présente une grande variété de paramètres utilisables : *In JDBC with MYSQL, what are the types of parameters that can be assigned in a PreparedStatment (methods such as setInt, setString...)

    L'instruction Statement.RETURN_GENERATED_KEYS permet de récupérer les clés primaires générées par MySQL lors de l'insertion. La requête peut donc être lue pour récupérer l'information.

    L'instruction executeUpdate (ligne 15) permet de faire une requête qui modifie la structure de la base données (C_UD).

    Try with resources

    La syntaxe try(...)est appelée try with resources et assure que la ressource est fermée avec l'interface java.io.Closeable à la fin du bloc, qu'une exception soit levée ou non. Si vous n'utilisez pas cette syntaxe vous devez appeler requete.close(); à la fin des opérations dans le bloc finally.

    try (Connection connexion = DriverManager.getConnection("jdbc:mysql://127.0.0.1:3306/nomBD", "root", "mysql")) {
    PreparedStatement requete = connexion.prepareStatement("SELECT id, destination, moyenTransport, ordre FROM Deplacement");
    ResultSet resultats = requete.executeQuery();

    ArrayList<DeplacementBrut> donneesDeplacements = new ArrayList<>();

    // Accumulation des données
    while (resultats.next()) {
        DeplacementBrut donneesDeplacement = new DeplacementBrut(
                resultats.getInt("id"),
                resultats.getInt("destination"),
                resultats.getInt("moyenTransport"),
                resultats.getInt("ordre")
        );
        donneesDeplacements.add(donneesDeplacement);
    }

 } catch (SQLException exception) {
    throw new Exception("Impossible de charger la collection de deplacement.");

    On remarque à la ligne 3 l'instruction executeQuery qui récupère les données de la base de données. Elle joue le même rôle que executeUpdate. La méthode retourne un ResultSet que l'on peut parcourir pour récupérer les données. Le ResultSet est structuré sur le modèle d'un itérateur.

    Pourquoi un objet DeplacementBrut ?

    En Java, lorsque deux objets sont dépendants, on affecte une référence d'un objet à l'autre. En base de donnée, on stocke plutôt une copie de la clé primaire. Donc lorsqu'on crée les objets du côté Java, il faut aussi charger les dépendances de la bd. Comme on ne peut pas avoir deux connexions simultanées, il faut stocker temporairement les relations sous forme de clé étrangère, puis après charger les dépendances. DeplacementBrut, comme classe locale, permet ed s'acquitter de cette tâche.

    Chacun des champs est chargé avec getInt, comme on charge des identifiants. On doit utiliser la méthode propre au type de données comme pour l'assignation vue à la section précédente.
