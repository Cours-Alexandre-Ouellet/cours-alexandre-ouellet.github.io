    Notation d'algorithmes

    Contenus de la page

    2-3 Esquisses d'algorithmes

    Temps requis

    120 minutes

    Plusieurs concepts d'algorithme et de logique propositionnelle sont communs à plusieurs langages de programmation. Pour représenter et faciliter le transfert d'un algorithme ou d'un concept entre des langages, on peut utiliser le pseudo-code ou les diagrammes de flux de données. Voici quelques exemples de transfert de concepts entre des langages de programmation.

    Diagrammes de flux de données

    Le diagramme de flux de données (simplement appelés diagramme de flux) sert à représenter graphiquement l'exécution d'un algorithme. Le diagramme définit 4 formes qui représentent chacune une étape élémentaire du traitement des données.

    Début et fin de l'algorithme
    Ce type de forme indique le point de début et de fin d'un algorithme. On peut écrire dans le cercle les termes "Début" ou "Fin"
    Entrée et sortie
    Les entrées et sorties sont les lectures et les affichages de données par le système. Dans le parallélogramme, on indique les contenus lus et, si cela s'applique, la variable qui contiendra l'information.
    Décision
    Indique un choix entre plusieurs chemins d'exécution. On se sert des décisions aussi bien pour les instructions conditionnelles que les boucles comme il sera présenté plus loin. On indique dans le losange la condition logique.
    Processus
    Les processus sont simplement toutes les opérations qui ne correspondent pas à une de celles nommées ci-dessus. On indique dans la boîte le traitement de données.
    Flèche
    Des flèches simples lient les différentes formes. Sauf pour la forme de décision, une seule flèche sort de chaque forme. Pour la forme de décision seulement, il faut indiquer sur la flèche les termes "oui" ou "non" pour indiquer si le chemin s'exécute lorsque la condition est satisfaite ou non.

    Qu'en est-il du else if ?

    Le else if se traite comme un else suivi d'un autre if donc un enchaînement de conditions.

        if(condition1) {
        // Traitement A
    } else if (condition2) {
        // Traitement B
    } else {
        // Traitement C
    }

    // Équivalent à 

    if(condition1) {
        // Traitement A
    } else {
        if (condition2) {
            // Traitement B
        } else {
            // Traitement C
        }
    }

    Les boucles

    Les boucles pour (for), aussi appelée jusqu'à et tant que (while) permettent de répéter un traitement selon une condition. Bien que strictement équivalentes du point de vue logique, chacune des boucles permet de faire le même traitement. La boucle pour chaque quant à elle, permet de parcourir une collection de données sans en modifier la structure.

    La borne supérieure

    Remarquez que dans le pseudo-code on inclut toujours la borne supérieure (jusqu'à), alors qu'il est commun de l'exlucre dans les langages de programmation.
