    Itérateurs et yield

    La notion d'itérateur se retrouve dans plusieurs langages de programmation. Les itérateurs permettent de contrôler la façon de parcourir une collection de données. En C#, ils sont implémentés par l'interface IEnumerator définie dans l'espace de noms System.Collection.

    public interface IEnumerator<T> 
{
    T Current;              // Élément pointé par l'itérateur (par défaut null)

    bool MoveNext();        // Déplace l'élément pointé sur le prochain. Retourne TRUE si c'est possible 
                            // et FALSE s'il n'y a pas d'autres éléments

    void Reset();           // Repositionne l'itérateur au début

    void Dispose();         // Libère les ressources de l'itérateur. Directement hérité de l'interface 
                            // IDisposable
}

public bool MoveNext()

    L'utilisation directe de l'interface IEnumerator par son implémentation dans un classe concrète (ou une lambda expression) peut devenir lourd et fastidieux. Il y a plusieurs méthodes à définir et généralement la logique d'itération est propre à une situation spécifique, ce qui amène à définir une nouvelle classe chaque fois que l'utilisation d'un itérateur est nécessaire.

    Le mot-clé yield de C# nous permet de définir implicitement un itérateur. Ce mot-clé permet d'indiquer à quel moment est-ce que l'itérateur doit avancer vers un nouvel élément.

    Prenons l'exemple de la classe AfficheurNombresCarres. Nous pourrions faire un itérateur qui avance à chaque fois qu'une touche est pressée. En utilisant yield on se retrouve avec une seule méthode qui est statique. Comme la méthode est statique, on pourrait, selon le contexte de l'application, déplacer ce code dans la classe utilisant la valeur des carrés.

    public static IEnumerator<int> GetSequence(ConsoleKey toucheContinuer = ConsoleKey.Tab)
{
    int nombre = 1;

    Console.WriteLine($"Appuyez sur {toucheContinuer} pour afficher des carres. Appuyez sur n'importe" +
    $"quelle autre touche pour arreter.");

    // Tant que la touche est la touche continuer
    while(Console.ReadKey().Key.Equals(toucheContinuer))
    {
        yield return nombre * nombre;           // Équivalent de MoveNext
        nombre++;
    }
}

    Donc le MoveNext permet d'exécuter la fonction jusqu'à la ligne qui contient le yield return. La valeur de Current est alors mise à jour et MoveNext retourne true. Si l'exécution de la fonction n'atteint pas la ligne yield avant la fin de la méthode(par exemple la condition du while n'est pas vérifiée), MoveNext retourne false.

    yield ou yield return ?

    On parle du mot-clé yield, mais en pratique yield n'apparaît jamais sans un return.

    Pourquoi utiliser des itérateurs ?

    Les itérateurs sont généralement utilisés dans trois contextes :

    1) Pour parcourir des collections non ordonnées comme un tas (HashMap, HashSet, Dictionary... ). 2) Pour éviter de générer une large collection de données. Comme on peut générer à chaque passage la valeur à utiliser, on ne garde jamais en mémoire plus d'une valeur. 3) Quand on veut contrôler le moment où les éléments sont reçus. Par exemple, en réseautique, on peut attendre une confirmation avant l'envoie du prochain segment de données. Un itérateur permet de parcourir une longue chaîne binaire en plus courtes séquences et d'attendre la confirmation de la précédente avant l'envoie de la suivante.

    Avec Unity, nous sommes vraiment dans le 3e cas d'utilisation où l'on veut une information par frame lorsqu'on utilise les itérateurs dans le contexte d'une coroutine.

    Coroutine Unity

    Dans le contexte d'une coroutine, on veut exécuter une méthode sur plusieurs frames. On doit donc y définir un point d'arrêt pour attendre la prochaine frame. On utilise alors un itérateur (qui ne parcours non pas une liste de données classique, mais une liste de frames). On manipule les coroutines de la façon suivante.

    Dans la coroutine, on peut utiliser 4 types de return

    1. return null ou return 0 : attend la prochaine frame de la coroutine avant de poursuivre l'exécution.
    2. return new WaitForSeconds(float) : attend pour le nombre de secondes écoulées dans le jeu.
    3. return new WaitForSecondsRealtime(float) : attend pour un nombre de secondes écoulées dans le monde réel.
    4. return new WaitUntil(lambda) ou return new WaitWhile(lambda) : attend selon une lambda tant qu'elle est fausse (until) ou tant qu'elle est vraie (while)

    // Attend une frame
yield return null;              

// Attend 3 secondes dans le jeu
yield return new WaitForSeconds(3.0f);

// Attend 5 secondes dans la vraie vie
yield return new WaitForSecondsRealtime(5.0f);

// Attend tant que la vie du personnage est strictement positive
yield return new WaitWhile(() => personnage.Vie > 0);

// Attend que le personnage meurt
yield return new WaitUntil(() => personnage.Vie <= 0);
