    Les fonctions

    Qu'est-ce qu'une fonction ?

    Les fonctions, d'un point de vue des mathématiques, sont des opérateurs complexes qui acceptent une ou plusieurs variables et retourne une ou plusieurs valeurs. Une fonction mathématique a comme seule contrainte que pour les mêmes valeurs acceptées, le même résultat est retourné (autrement dit chaque valeur du domaine est associée à une seule valeur de l'image de la fonction). On peut penser aux fonctions polynomiales ou trigonométriques classiques (\(f(x) = x\), \(f(x) = 5\) ou \(f(x, y) = x+y\)).

    En informatique, le terme fonction désigne une idée similaire : soit le fait de prendre des valeurs et d'appliquer des opérations sur ces valeurs. Les fonctions sont des blocs de code qui encapsulent un traitement. Ces blocs de code acceptent aucune, une ou plusieurs variable et retourne aucun ou un seul résultat. Dans le code, on retrouve une instruction qui vise à déclarer (declare) une fonction et une instruction qui vise à appeler ou invoquer (call ou invoke) la fonction.

    Déclarer une fonction

    Le mot-clé def est utilisé pour déclarer un fonction. On indique le nom de la fonction (mème règles d'indicateur que pour les variables). On place ensuite entre parenthèses les paramètres de la fonction. Si la fonction n'a pas de paramètres, on place une paire de parenthèses vides. On place ensuite un deux-points puis tout le code de la fonction sera identé d'un niveau.

    Retourner des valeurs

    Il est possible de retourner une valeur à la fin de l'exécution d'une fonction. Pour se faire, on utilise l'instruction return. Dès que cette instruction est rencontrée, l'exécution de la fonction s'arrête et la valeur est retournée. Il est possible de mettre plusieurs instructions return dans une fonction. Dans ce cas, la première instruction return exécutée met fin à l'exécution de la fonction.

    Portée des variables

    Les variables déclarées dans les fonction et les paramètres ont une portée scope limitée à la fonction qui les déclare. Autrement dit, les variables et les paramètres suivent ces deux comportements :

    • On peut réutiliser les identificateurs utilisés ailleurs sans risque d'effet de bord (que la modification d'une ait un impact sur l'autre); et
    • On ne peut pas utiliser les valeurs des paramètres et variables déclarées dans la fonction hors de celle-ci.

    Ordre de déclaration des fonctions

    Les fonctions doivent être déclarées avant d'être utilisées. C'est pourquoi on place toujours la les lignes suivantes à la fin du fichier.

    if __name__ == "__main__":
    main()

    Valeurs par défaut

    Il est possible définir des valeurs par défaut à des paramètres de la fonction. Les valeurs par défaut font en sorte qu'il n'est pas nécessaire d'indiquer le paramètre lors de l'appel. Pour indiquer une valeur par défaut, on ajoute un symbole d'égalité après le paramètre suivi de la valeur défaut. Par exemple, voici une définition de la fonction calcul_taxes avec des valeurs par défaut.

    Indiquer explicitement les paramètres lors de l'appel

    Normalement, on appelle la fonction avec les paramètres dans le même ordre que lors de la déclaration. On peut changer cet ordre en indiquant le nom des paramètres lors de l'appel. Cette technique permet de contourner le problème rencontré lorsqu'on veut utiliser que certaines des valeurs par défaut. Pour indiquer explicitement les paramètres lors de l'appel, on indique dans les parenthèses lors de l'appel le nom du paramètre suivi d'un symbole d'égalité et de sa valeur.

    Solution

    1. Pour trouver les zéros d'une fonction, on peut utiliser la méthode de Newton. Il s'agit de construire une suite de points convergents (ou du moins ils convergent généralement ) vers l'un des zéros de la fonction. À l'aide de la méthode de Newton, trouvez le zéro réel, avec 4 décimales de précision, de la fonction \(f(x) = x^3-3x^2+4x-4\) dont la dérivée est \(f'(x) = 3x^2-6x+4\) (réponse 2). Les termes itérés de la méthode de Newton prennent la forme suivante :

    $$ x_{n+1} = x_n - f(x_n)/f'(x_n) $$

    x = 0.0         # Point de départ la solution
ecart = 1.0     # Écart entre les deux dernières solutions

while ecart > 0.0001 or ecart < -0.0001:
    # ecart verifie l'avancement vers la solution
    fonction = x**3-3*x**2+4*x-4
    derivee = 3*x**2-6*x+4

    # Gere les erreurs de division par 0
    if(derivee == 0.0):
        ecart = 0
        print("Division par 0. Impossible d'evaluer avec la valeur intiale " + str(x))
    else:
        terme = x - fonction / derivee
        # Calcule la différence entre les deux derniere solution
        ecart = terme - x
        # Le changement de x par la nouvelle valeur (terme) prepare pour la prochaine iteration
        x = terme
