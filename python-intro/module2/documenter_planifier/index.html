    Documenter le code

    Plus le code augmente en taille, plus il est important de laisser des traces des réflexions et de l'utilité du code. C'est ce que l'on appelle la documentation. L'utilisation de la documentation permet de lire en texte ce qu'une fonction fait sans constamment devoir en relire le code. Elle aide aussi à reconnaître l'intention derrière une fonction ou les astuces utilisées pour contourner certains problèmes.

    Commentaires de code

    Les commentaires dans le code prennent la forme d'une simple phrase placé après le caractère #. On peut diviser le commentaire sur plusieurs lignes en répétant le symbole # au début de chaque ligne. Il n'est pas nécessaire de commenter chaque ligne (en fait même ça n'apporte rien et allourdit la lecture). On réserve les commentaires pour les lignes plus complexes ou celles qui introduisent un bloc de code (par exemple, un boucle qui aurait 3 instructions à l'intérieur pourrait avoir un commentaire qui la précède.)

    Docstring

    Les docstring sont des commentaires qui expliquent ce qu'une fonction fait. Elles consistent en plusieurs lignes délimitées par 3 guillemets anglais. Aucun retour à la ligne n'est fait avant ou après la docstring.

    Exemple de documentation d'une fonction
        def aire_rectangle(base, hauteur = 1.0):
        """Calcul l'aire d'un rectangle et la retourne.

        Keywords arguments:
        base -- la base du rectangle
        hauteur -- la hauteur du rectangle (default = 0.0)
        """
        return base * hauteur

    Expressions anglaises

    Il est important d'utiliser l'expression anglaise « Keywords arguments: » et « default » pour introduire les paramètres de la fonction et les valeurs par défaut, car certains traitements peuvent utiliser les docstrings et utiliseront cette expression.

    Exercice - premier programme complet

    Planifiez et programmez une calculatrice simple. La calculatrice doit permettre d'additionner, de soustraire, de multiplier et de diviser des nombres. Chaque fois qu'un calcul est effectué, la réponse est affichée à la personne utilisatrice. Tant que la personne utilisatrice ne demande pas de quitter, la calculatrice doit demander un nouveau calcul.
