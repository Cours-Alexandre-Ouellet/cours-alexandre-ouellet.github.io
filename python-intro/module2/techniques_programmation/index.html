    Diviser son code entre plusieurs fichiers

    Lorsque les programmes grossissent, il devient intéressant de diviser son code entre plusieurs fichiers pour deux raisons :

    Augmenter la lisibilité du code

    Il est facile de se perdre dans de longs fichiers de code. La division des fonctions entre plusieurs fichiers plus courts augmente la lisibilité et permet de rassembler les fonctions qui partagent un but semblable.

    La raison du code qui appelle main

    Le bout de code suivant 

    if __name__ == "__main__":
    main()

    sert entre autres à éviter d'exécuter plusieurs fois le programme à cause d'inclusions multiples de fichiers. Donc lorsqu'on crée du code avec plusieurs fichier, il est important d'inscrire ce bout de code et de ne pas créer du code qui n'est pas dans une fonction, sinon ce code sera exécuté à chaque inclusion du fichier.

    Augmenter la réutilisation du code
    Si le code est dans un fichier avec seulement des fonctions qui partagent un but semblable (ex. fonctions physiques pour le mouvement rectiligne, fonctions de chimie pour les gaz, fonctions de géométrie ...), c'est plus simple de réutiliser ce code dans plusieurs programmes. Cela réduit la quantité de code à réécrire entre plusieurs projets.

    Pour de petits projets, on peut placer tous les fichiers dans le même dossier. Il faut cependant indiquer explicitement à Python quoi importer.

    Importer un fichier

    Prenons l'exemple d'un projet de géométrie avec deux fichiers : un fichier geometrie.py qui définit des fonctions et un fichier main.py qui interagit avec la personne utilisatrice. On appelle un fichier un module d'un point de vue du code.

    Pour importer les fonctionnalités du module geometrie vers main, il faut utiliser l'instruction #!import. Cette instruction a différente variation selon le résultat recherché.

# Importation d'une fonction avec un alias
from geometrie import distance as geo_dist  #, ...
distance = geo_dist(...)

    Instruction d'importation

    Les instructions pour importer d'autres fichiers sont toujours les premières (avant la déclaration des fonctions).

    Pourquoi importer juste une fonction plutôt que toujours tout le contenu du module ?

    Pour éviter les collisions de noms. Si deux modules déclarent une fonction de même nom, on peut les distinguer si l'on doit préfixer l'appel du nom du module. Sans cette distinction, Python ne saura laquelle des versions appeler.

    Vérifier et déboguer son code

    Comment savoir si son code fonctionne vraiment ? Ça demande une preuve mathématique ce qu'en bon informaticien paresseux on ne fait ... jamais (ou presque). On effectue plutôt des tests et la seule chose que les tests peuvent révéler ce sont des cas d'erreur. Les tests ne permettent pas de s'assurer qu'un code ne contient pas d'erreur. Les stratégies d'assurance qualité mise en place sont :

    • Tester plusieurs scénarios différents :
    • Valeurs qui s'exécutent correctement;
    • Valeurs qui provoquent des erreurs (division par zéro, dépasse d'indice, valeur négative alors qu'elle doit être positive);
    • Valeurs limites ou inusités (exemple une classe avec 157 843 401 élèves !)
    • Faire relire son code par une tierce personne
    • Réaliser les tests de façon automatique de façon à pouvoir les réutiliser (tests de régression)
    • Mettre l'accent sur les composants critiques, les tester avec et sans interactions

    Une fois les tests exécutés et les cas fautifs révélés, il faut trouver la source de l'erreur. Dans certains cas simples, c'est un travail d'observation avec une feuille de papier et un crayon qui permet rapidement d'identifier le problème. D'autres fois, il faut avoir recours au débogueur pour venir à bout de comprendre ce que le code que l'on a écrit fait.

    Sources d'informations

    Pour continuer avec Python, plusieurs sources d'informations sont disponibles :

    1. Documentations officielles :

      • Documentation de Python
      • Documentation des styles de programmation en Python

    2. Sites web :

      • W3Schools qui propose souvent des condensés des fonctions Python. Utile pour une référence de syntaxe rapide, mais omet nombre d'informations importantes.
      • OpenClassroom propose un cours de Python de base, ainsi que plusieurs cours sur des sujets connexes en Python. L'ensemble des contenus est disponible en français.
      • StackOverflow plus grand forum de programmeur au monde. Beaucoup de recherches faites dans Google aboutissent sur StackOverflow. Les réponses sont soumises à un système de votation et la plupart des contributeurs sont très compétents dans leur domaine.
      • Developpez est un grand forum de programmeurs de la francophonie. Il offre aussi quelques didacticiels sur le langage Python.

    3. Utilisation des intelligences artificielles :

      ChatGPT et Copilot fournissent souvent des codes Python de qualité acceptable qui produisent généralement le bon résultat.
