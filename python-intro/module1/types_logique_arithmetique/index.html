    Types de données, opérations et conditions logiques

    Objectifs de la section

    Manipuler des données dans les types natifs
    Effectuer un traitement conditionnel
    Exprimer des conditions logiques à l'aide d'expressions booléennes composées

    Temps requis

    50 minutes

    Tel que vu à la section précédente, il existe différents types aux données que l'on manipule. Les fonctions et les opérations s'attendent à recevoir des données dans certains types, il faut donc s'assurer de les respecter. Le langage python est dit typé :

    • fort : on ne peut pas mélanger les types de données, il faut faire les conversions explicites; et
    • dynamique : on ne précise pas le type dans la déclaration de la variable, il est déduit par l'interpréteur de python.

    Concrètement, cela signifie que l'on ne peut pas par exemple additionner une chaîne de caractères et un entier, mais on peut assigner une chaîne de caractères à une variable, puis remplacer cette valeur par un entier, et ce, dans la même variable.

    Types de données

    On trouve en python les types de données suivants :

    • Nombres entiers;
    • Nombres à virgule flottante;
    • Chaînes de caractères;
    • Booléens (variables logiques);
    • Tableaux (vus au module 2);
    • Structures de données (vues au module 2); et
    • Classes (pas vues dans la formation).

    Nombres entiers

    Les nombres entiers sont représentés en format binaire (composé de 0 et de 1). Le binaire fonctionne comme le système décimal, mais au lieu de compter en base 10, on compte en base 2.

    Par exemple le nombre 101 en décimale est en fait la somme suivante : \(101 = 1 \times 10^2 + 0 \times 10^1 + 1 \times 10^0\)

    Le même nombre en binaire représente plutôt la somme suivante : \(101 = 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0\) ... soit la valeur 5 en base décimale.

    Les entiers en python, contrairement à bien des langages de programmation, sont encodés sur un nombre de bits variable, ce qui évite les erreurs de débordement (overflow) constatées avec d'autres langages.

    Nombres à virgule flottante

    Les nombres à virgule (appelés virgule flottante en informatique) sont également représentés en binaire, à l'aide d'un masque. Le passage de la base décimale en la base binaire se fait moins naturellement que pour les entiers à cause du changement des nombres à représentation périodique.

    Par exemple en base 10, on écrit souvent la fraction \(\frac{1}{3}\) comme 0.33 ou 0.333 ... selon le degré de précision souhaité. Cet arrondi cause des erreurs arithmétiques qui s'accumulent lors des calculs numériques. En binaire, le même phénomène se produit, mais sur des valeurs différentes. Par exemple la fraction \(\frac{1}{10}\) est représentée en binaire par 0.0011 0011 0011 (0011) répété à l'infini, donc est aussi arrondi et cause des erreurs arithmétiques. Par exemple, \(\frac{1}{10} + \frac{1}{10} + \frac{1}{10} = 0.30000000000000004\).

    De plus, la représentation en virgule flottante exploite la notation scientifique. Donc la précision est autour de la virgule (d'où le nom virgule flottante) et il est possible de perdre une partie de l'information pour les grands nombres. Python présente que les 17 premières décimales d'un nombre, même si plus sont calculées. Les nombres décimaux doivent être 2.2250738585072014e-308 entre 1.7976931348623157e+308.

    Représentation des types logiques

    On utilise souvent en informatique des variables booléennes, c'est-à-dire qu'elles peuvent prendre que la valeur vraie (True) ou fausse (False). Ce type de valeurs est très utile pour se traiter un état ou un paramètre d'un système. On peut aussi s'en servir dans des expressions logiques afin de construire des expressions complexes.

    La majuscule

    Il est important de porter attention à la majuscule, car le langage Python est sensible à la casse. Par exemple, true provoque une erreur, tandis que True est une expression valide.

    Représentation de rien pantoute

    Il peut arriver que l'on doive déclarer une variable, mais qu'on ne connaisse pas le contenu de celle-ci au moment de la déclaration. On peut utiliser le mot-clé None pour indiquer que son contenu est vide.

    Déclaration d'une variable vide
    1
    variable = None

    Conversion entre les types

    On peut faire des conversions entre les types, par exemple transformée une chaîne de caractère lue en un entier. C'est ce qui a été fait à l'exercice de la section précédente où l'on appelait la fonction int sur une chaîne de caractères. Les fonctions sont toujours les mêmes que les types de données :

    • Entier : int
    • Virgule flottant : float
    • Booléen : bool
    • Chaîne de caractères : str

    Conversion vers None

    Il est impossible de convertir vers ou à partir du type None : il n'y a pas de données à convertir.

    Instruction conditionnelle

    L'une des caractéristiques les plus importantes des programmes est de réagir de façon différenciée aux différentes entrées. Comme présenté au début du module, l'une des caractéristiques des ordinateurs est d'accepter des données, il faut pouvoir les considérer dans les traitements effectués.

    L'idée de la clause conditionnelle est d'effectuer une vérification sur un booléen et d'effectuer un traitement différent selon la valeur.

    graph LR
  A[Code...] --> B{Test logique};
  B -->|Faux| D[Code...];
  B -->|Vrai| C[Traitement conditionnel];
  C --> D;

    Dans le code python, cela prend la forme d'un condition !#py if.

    L'indentation

    Remarquez l'indentation (retrait de la marge de gauche) supplémentaire sous le !#py if pour la section traitement conditionnel. C'est ce qui permet à Python de savoir quelles instructions sont dans le bloc d'instructions conditionnelles (il peut y avoir plusieurs lignes) et quand celui-ci se termine.

    Il y a trois façons de construire des tests logiques pour des expressions conditionnelles :

    • Utiliser une variable booléenne;
    • Utiliser un comparateur (on voit ça à la prochaine sous-section); et
    • Utiliser une fonction qui retourne un booléen (on voit ça au prochain module);

    Comparateurs

    Les comparateurs, comme leur nom l'indique permettent de comparer deux valeurs ensemble. Il existe 6 comparateurs dont la signification peut varier selon le type de données.

    Comparateur Symbole Interprétation des types numériques Interprétation du type booléen Interprétation des chaînes de caractères
    Égalité == Vérifie si c'est exactement la même valeur (très sensible aux erreurs arithmétiques des nombres à virgule flottante). Vérifie si les deux booléens ont la même valeur. Vérifie si le contenu des chaînes est le même (sensible à la casse).
    Différence != Vérifie si c'est deux valeurs différentes (très sensible aux erreurs arithmétiques des nombres à virgule flottante). Vérifie si les deux booléens n'ont pas la même valeur. Vérifie si le contenu des chaînes est le même (sensible à la casse).
    Plus petit que < Vérifie que la valeur d'un est strictement plus petite que l'autre. False est plus petit que True. Compare en ordre lexicographique croissant selon la table ASCII.
    Plus petit que ou égal <= Vérifie que la valeur d'un est plus petite ou égale à l'autre. False est plus petit ou égal à True. Compare en ordre lexicographique croissant selon la table ASCII.
    Plus grand que > Vérifie que la valeur d'un est strictement plus grande que l'autre. True est plus grand que False. Compare en ordre lexicographique décroissant selon la table ASCII.
    Plus grand que ou égal >= Vérifie que la valeur d'un est plus grande ou égale à l'autre. True est plus grand ou égal à False. Compare en ordre lexicographique décroissant selon la table ASCII.

    Et sinon encore...

    Il arrive qu'on puisse avoir plusieurs scénarios d'exécution possibles qui sont exclusifs entre eux.

    graph LR
  A[Code...] --> B{Test logique};
  B --> C[Traitement conditionnel];
  E --> D[Code...];
  B --> E[Traitement alternatif 1];
  B --> F[Traitement alternatif 2] --> D;
  B --> G[Traitement alternatif ...] --> D;

    On utilise dans ce cas les clauses elif (contraction de else if) qui acceptent aussi une condition. Il peut y avoir plusieurs clauses elif qui s'enchaîne. La première clause dont la condition est évaluée à vraie est exécutée, puis le programme continu. La clause if est toujours la première et la clause else, lorsqu'elle est nécessaire, est toujours la dernière.

    Opérations logiques

    Les opérateurs entre les variables booléennes sont différents de ceux des variables numériques. On trouve les 3 opérations suivantes :

    • Disjonction (ou, or) : opérateur binaire faux seulement si les deux opérandes sont fausses;
    • Conjonction (et, and) : opérateur binaire vrai seulement si les deux opérandes sont vraies; et
    • Négation (non, not) : opération unaire qui inverse la valeur de l'opérande.

    On représente souvent les combinaisons possibles à l'aide de tables de vérité.

    Table de vérité de la disjonction (or)

    Opérande 1 Opérande 2 Résultat
    False False False
    False True True
    True False True
    True True True

    Table de vérité de la conjonction (and)

    Opérande 1 Opérande 2 Résultat
    False False False
    False True False
    True False False
    True True True

    Concepts clés de la section

    booléen et opérateurs logiques
    les booléens représentent une information qui peut prendre deux états vrai ou faux. On peut combiner plusieurs expressions booléennes par les opérateurs et et ou. On peut aussi inverser un état avec l'opérateur de négation.
    comparateur
    vérifie une relation entre deux expressions et retourne une valeur booléenne selon si la relation est vérifiée au non.
    conversion de type
    permet de changer la représentation typée d'une donnée entre deux types. Elle peut provoquer des erreurs ou des pertes d'information.
    instruction conditionnelle
    permet d'exécuter une partie du code seulement lorsqu'une condition est vérifiée. On peut préciser des traitements alternatifs si la situation le requiert.

    Les fonctions et les mots-clés vus dans cette section

    and
    vérifie deux expressions booléennes et retourne un booléen vrai seulement si les deux expressions sont vraies.
    bool
    permet de convertir une donnée vers le type booléen.
    False
    valeur booléenne fausse.
    float
    permet de convertir une donnée vers une représentation en virgule flottante.
    if - elif - else
    mots-clés pour définir les instructions conditionnelles.
    int
    permet de convertir une donnée vers une représentation entière en base 10.
    str
    permet de convertir une donnée vers une représentation en chaîne de caractères.
    None
    représente l'absence de type de données.
    or
    vérifie deux expressions booléennes et retourne un booléen vrai seulement dès que l'une ou l'autre des expressions est vraie.
    True
    valeur booléenne vraie.
