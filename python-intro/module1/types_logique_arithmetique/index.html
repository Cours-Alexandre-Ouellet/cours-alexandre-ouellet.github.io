    Types de données, opérations et conditions logiques

    Tel que vu à la section précédente, il existe différents types aux données que l'on manipule. Les fonctions et les opérations s'attendent à recevoir des données dans certains types, il faut donc s'assurer de les respecter. Le langage python est dit typé : - fort : on ne peut pas mélanger les types de données, il faut faire les conversions explicites; et - dynamique : on ne précise pas le type dans la déclaration de la variable, il est déduit par l'interpréteur de python.

    Concrètement, cela signifie que l'on ne peut pas par exemple additionner une chaîne de caractères et un entier, mais on peut assigner une chaîne de caractères à une variable, puis assigner un entier dans la même variable

    Types de données

    On trouve en python les types de données suivants :

    • Nombres entiers;
    • Nombres à virgule flottante;
    • Chaînes de caractères;
    • Booléens (variables logiques);
    • Tableaux (vus au module 2);
    • Structures de données (vues au module 2); et
    • Classes (pas vues dans la formation).

    Nombres entiers

    Les nombres entiers sont représentés en format binaire (composé de 0 et de 1). Le binaire fonctionne comme le système décimal, mais au lieu de compter en base 10, on compte en base 2.

    Par exemple le nombre 101 en décimale est en fait la somme suivante : \(101 = 1 \times 10^2 + 0 \times 10^1 + 1 \times 10^0\)

    Le même nombre en binaire représente plutôt la somme suivante : \(101 = 1 \times 2^2 + 0 \times 2^1 + 1 \times 2^0\) ... soit la valeur 5 en base décimale.

    Les entiers en python, contrairement à bien des langages de programmation, sont encodés sur un nombre de bits variable, ce qui évite les erreurs de débordement (overflow) constatés avec d'autres langages.

    Nombres à virgule flottante

    Les nombres à virgule (appelés vrigule flottante en informatique) sont également représentés en binaire, à l'aide d'un masque. Le passage de la base décimale en la base binaire se fait moins naturellement que pour les entiers à cause du changement des nombres à représentation périodique.

    Par exemple en base 10, on écrit souvent la fraction \(\frac{1}{3}\) comme 0.33 ou 0.333 ... selon le degré de précision souhaité. Cet arrondi cause des erreurs arithmétiques qui s'accumulent lors des calculs. En binaire, le même phénomène se produit, mais sur des valeurs différentes. Par exemple la fraction \(\frac{1}{10}\) est représentée en binaire par 0.0011 0011 0011 (0011) répété à l'infini, donc est aussi arrondit et cause des erreurs arithmétiques. Par exemple, \(\frac{1}{10} + \frac{1}{10} + \frac{1}{10} = 0.30000000000000004\).

    De plus, la représentation en virgule flottante exploite la notation scientifique. Donc la précision est autour de la virgule (d'où le nom virgule flottante) et il est possible de perdre une partie de l'information pour les grands nombres. Python présente que les 17 premières décimales d'un nombre, même si plus sont calculées. Les nombres décimaux doivent être 2.2250738585072014e-308 entre 1.7976931348623157e+308.
